# PRD: AI 업무 자동화 에이전트 (General Work Agent)

## 1. 제품 개요

### 1.1 배경
현재의 코딩 에이전트는 코드 생성·수정·검증에 강점이 있으나, 실제 조직의 업무 흐름(문서, 메일, 회의, 일정, 결재, 대외 커뮤니케이션)까지는 충분히 커버하지 못한다. 본 PRD는 코딩 에이전트를 확장하여 **일반 업무 전반을 처리하는 자동화 에이전트**의 기본 설계를 정의한다.

### 1.2 목적
- 반복적인 사무 업무의 자동화
- 업무 품질의 표준화 및 누락 방지
- 승인·보안·감사 요구사항을 기본 탑재한 안전한 자동화
- 개발자뿐 아니라 비개발자도 활용 가능한 범용 에이전트 제공

### 1.3 핵심 가치 제안
> “AI가 문서를 ‘써주는 것’이 아니라, **업무를 끝까지 처리하는 주체**가 된다.”

---

## 2. 목표 사용자

- 공공/기업 조직의 실무자
- 기획자, 행정직, PM, 팀장
- 개발자 및 기술 리드(업무+기술 병행)
- DevOps / 플랫폼 운영 인력

---

## 3. 주요 사용 시나리오

1. 회의 메모를 입력하면
   - 회의록 정리
   - 결정사항/액션아이템 추출
   - 후속 메일 초안 생성
   - 일정 등록 초안 생성
2. 업무 요청(“이 내용으로 대외 회신 작성”)을 입력하면
   - 조직 템플릿 적용
   - 톤/표현 정규화
   - 대외 발신 승인 요청
3. 데이터 파일을 주면
   - 집계/지표 계산
   - 표/차트 생성
   - 결과 보고서 초안 작성

---

## 4. 시스템 아키텍처 개요

### 4.1 구조 개념
- Orchestrator(Planner): 업무 분해·위임·상태 관리
- Sub-Agents: 도메인별 전문 업무 처리
- Tool Layer: 문서/메일/일정/데이터 등 실제 행동 수행
- Hook Layer: 규칙·보안·승인·품질 강제
- Rule System: 조직/프로젝트별 업무 규칙 주입

---

## 5. 에이전트 구성

### 5.1 Orchestrator (필수)
- 사용자 요청을 업무 단위로 분해
- 적절한 서브 에이전트에 위임
- 결과 수합 및 완료 판단

### 5.2 Sub-Agent 역할

| 에이전트 | 주요 역할 |
|---|---|
| Doc-Agent | 보고서, 회의록, 제안서 작성 |
| Comms-Agent | 메일, 공지, 대외 커뮤니케이션 |
| PMO-Agent | 일정, 액션아이템, 결재 흐름 |
| Data-Agent | 엑셀/지표/집계/차트 |
| Research-Agent | 근거 조사 및 요약 |
| Compliance-Agent | 보안·개인정보·규정 점검 |
| Ops-Agent | 파일 관리, 공유, 등록 |

---

## 6. 핵심 기능 요구사항

### 6.1 업무 분류 및 라우팅
- 입력 요청을 자동으로 업무 유형 분류
- 민감도 태그(대외/개인정보/보안) 자동 부착

### 6.2 워크플로우 관리
- 상태 머신 기반 업무 흐름
  - DRAFT → REVIEW → APPROVAL → COMPLETED
- 각 단계별 필수 조건 정의

### 6.3 승인(Approval) 시스템
- 대외 발신, 일정 등록, 파일 공유, 개인정보 포함 시 승인 필수
- 승인 이력 및 로그 저장

### 6.4 규칙(Rules) 시스템
- 조직별 업무 규칙 파일 로드
- 템플릿, 말투, 보안 정책 자동 주입

### 6.5 툴 실행
- 문서 생성/편집
- 메일 초안 생성
- 일정 후보 생성
- 데이터 집계 및 시각화

---

## 7. 훅(Hook) 기반 강제 정책

### 7.1 주요 훅
- chat.message: 업무 분류/리스크 태깅
- system.transform: 규칙·템플릿 주입
- tool.execute.before/after: 품질 게이트
- permission.ask: 승인 강제
- command.execute.before: 위험 작업 차단

### 7.2 강제 원칙
- 승인 없는 대외 발신 금지
- 수치 포함 문서는 출처·기간 명시
- 회의록에는 액션아이템 필수 포함

---

## 8. 완료 기준 (Definition of Done)

업무는 다음을 충족해야 완료로 간주된다.
- 산출물 생성 완료
- 규칙 및 템플릿 준수
- 필요한 승인 획득
- 최종 요약 및 다음 액션 제시

---

## 9. 비기능 요구사항

### 9.1 보안
- 개인정보 자동 탐지 및 마스킹
- 기본 deny + 명시적 승인

### 9.2 확장성
- 새로운 업무 에이전트 추가 가능
- 조직/프로젝트별 규칙 독립 적용

### 9.3 투명성
- 어떤 규칙·훅이 적용되었는지 설명 가능

---

## 10. MVP 범위

- Orchestrator + Doc/Comms/PMO Agent
- 승인(ask) 시스템
- 업무 규칙 파일(.rules) 로드
- 회의 → 회의록/메일/일정 자동화

---

## 11. 성공 지표 (Metrics)

- 승인 누락 0건
- 업무 처리 시간 단축률
- 재작업(수정 요청) 감소율
- 사용자 만족도

---

## 12. 요약

본 PRD는 코딩 에이전트를 넘어, **조직의 실제 업무를 안전하고 일관되게 처리하는 AI 에이전트**를 목표로 한다. 핵심은 자동화 그 자체가 아니라, 규칙·승인·품질이 보장된 업무 완료이다.
